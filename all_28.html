属性@property的使用

为什么要使用getter和setter方法

在绑定属性时，如果可以直接设置属性，虽然写起来很简单，但是没有办法检查参数，并且可以随意更改参数。

s = People()
s.salary = -1000

这显然不符合逻辑。为了限制salary的范围，可以通过一个setSalary方法来设置工资，再通过一个getSalary来获得工资，这样在setSalary方法里，就可以检查参数。

私有属性添加getter和setter方法

class People(object):
    def __init__(self):
        self.__salary = 0
    def getSalary(self):
        return self.__salary
    def setSalary(self, value):
        if not isinstance(value, int):
            raise ValueError('Money must be an integer!')
        if value <0 or value > 10000:
            raise ValueError('Money must between 0~10000!')
        
        self.__salary = value

注意点

t.num到底是调用getter还是setter，要根据实际的场景来判断，

  1. 如果是给t.num赋值，那么一定调用setNum()
  2. 如果是获取t.num的值，那么久一定调用getNum()

@property装饰器

但是直接调用getter()和setter()比较复杂。Python内置的@property装饰器可以把这两种方法转化为属性进行调用。看一下例子：

class People(object):
    def __init__(self):
        self.__salary = 0
    
    @property
    def salary(self):
        return self.__salary
    
    @salary.setter
    def salary(self, value):
        if not isinstance(value, int):
            raise ValueError('Salary must be an integer!')
        if value <0 or value > 10000:
            raise ValueError('Salary must between 0~10000!')
        
        self.__salary = value
        
    @property
    def tax(self):
        self.__tax = self.__salary*0.1
        return self.__tax

当不定义setter方法时，那么就定义了一个只读属性(例子中tax属性是只读属性)。

@property 成为属性函数，可以对属性赋值时做必要的检查，并保证代码的清晰短小，主要有2个作用：

  1. 将方法转换为只读
  2. 重新实现一个属性的设置和读取方法，可做边界判定

作用域

命名空间

命名空间就是生效的范围

全局，局部

查看局部变量

>>>> locals()

查看全局变量

>>>> globals()

LEGB规则

python 使用LEGB的顺序来查找一个符号对应的对象

locals-> enclosing function->globals->builtins

  • locals, 当前所在命名空间（如函数、模块），函数的参数也属于命名空间中的变量
  • enclosing，外部嵌套函数的命名空间（闭包中常见）
  • globals, 全局变量，函数定义所在模块的命名空间
  • builtins, 内建模块的命名空间

数据结构介绍

概念

数据是一个抽象的概念，将其进行分类后得到程序设计语言中的基本类型。如：int，float，char等。数据元素之间不是独立的，存在特定的关系，这些关系便是结构。数据结构指数据对象中数据元素之间的关系。

程序= 数据结构+算法
总结：算法是为了解决实际问题而设计的，数据结构是算法需要处理的问题载体。

抽象数据类型(Abstract Data Type)

抽象数据类型(ADT)的含义是指一个数学模型以及定义在此数学模型上的一组操作。即把数据类型和数据类型上的运算捆在一起，进行封装。引入抽象数据类型的目的是把数据类型的表示和数据类型上运算的实现与这些数据类型和运算在程序中的引用隔开，使它们相互独立。

内存、类型本质、连续存储

内存是连续的存储空间，内存的基本存储单元是一个字节，1个字节(Byte)是8位(Bit)

顺序表的内存形式：

  • 逻辑地址：0,1,2,3,...,n01
  • 元素存储: e0,e1,e2,e3,....,en-1
  • 物理地址:l0,l0+c,l0+2c,l0+3c,l0+(n-1)c

调试

pdb

pdb是基于命令行的调试工具，是Python的调试器，非常类似GNU的GDB（调试c/c++）

python -m pdb some.py # -m是模块的意思, pdb模块

命令

  • 输入命令l (list) 查看代码
  • 输入命令n (next) 可以单步执行代码
  • 输入命令c (continue) 继续执行代码
  • 输入命令b (break) 添加断点
  • 输入命令clear 清除断点
  • 输入命令p (print) 打印一个变量的值
  • 输入命令s (step) 进入到一个函数
  • 输入命令a (args) 打印所有的形参数据
  • 输入命令r (return) 快速执行到函数的最后一行
  • 输入命令q quit 退出调试
