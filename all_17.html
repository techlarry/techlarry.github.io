逻辑删除

逻辑删除：当数据非常重要时，在实际执行删除操作时，只是将数据标记为删除。在数据查询操作时，加上是否删除语句。

mysql> alter table birds add column isDelete bit default b'0';
mysql> update birds
    -> set isDelete = b'1'
    -> where bird_id = 101;
mysql> select count(*)
    -> from birds
    -> where `isDelete` = 0;
+----------+
| count(*) |
+----------+
| 28890    |
+----------+

Regular Expression in MySQL

MySQL support pattern-matching operations performed with the REGEXP operator in SQL statements.

mysql > select common_name as 'Birds Great and Small'
      -> from birds
      -> where common_name regexp 'Least|Great'
      -> order by common_name desc
      -> limit 10;

+-----------------------+
| Birds Great and Small |
+-----------------------+
| Least Tern            |
| Least Storm-Petrel    |
| Least Seedsnipe       |
| Least Sandpiper       |
| Least Pygmy-Owl       |
| Least Pauraque        |
| Least Nighthawk       |
| Least Honeyguide      |
| Least Grebe           |
| Least Flycatcher      |
+-----------------------+

REGEXP Operator is case insensitive. If we want an expression to be case sensitive, we'll need to add the Binary option.

mysql > select common_name as 'Hawks'
      -> from birds
      -> where common_name regexp binary 'Hawk'
      -> and common_name not regexp 'Hawk-Owl'
      -> order by family_id
      -> limit 10;

Syntax of Regular Expressions

  • ^: Match the beginning of a string
  • $: Match the end of a string
  • .: Match any character
  • a*: Match any sequence of zero or more a characters
  • a+: Match any sequence of one or more a characters
  • a?: Match either zero or one a character
  • de|abc: Match either of the sequences de or abc
  • (abc)*: Match zero or more instances of the sequence abc

InnoDB

MySql 数据库

  • E-R 关系模型，
  • 三范式
  • MySQL 数据类型
  • 约束

数据库（Database）是按照数据结构来组织、存储和管理数据的仓库，每个数据库都有一个或多个不同的API用于创建，访问，管理，搜索和复制所保存的数据。

mysql数据库，是当前应用非常广泛的一款关系型数据库。

数据库系统解决的问题：持久化存储，优化读写，保证数据的有效性

当前使用的数据库，主要分为两类：

  • 文档型，如sqlite, 就是一个文件，通过对文件的复制完成数据库的复制
  • 服务型，如mysql,postgre, 数据存储在一个物理文件中，但是需要使用终端以tcp/ip协议连接，进行数据库的读写操作 （有服务端、客户端，客户端连接服务端）

E-R 关系模型，

当前物理的数据库都是按照E-R模型进行设计的。

  • E表示Enty, 实体
  • R表示relationship, 关系
  • 一个实体转换为数据库中的一个表

  • 关系描述两个实体之间的对应规则，包括

    • 一对一
    • 一对多
    • 多对多

  • 关系转换为数据表中的一个利

  • 在关系型数据库中一行就是一个对象。

三范式

经过研究和对使用中问题的总结，对于设计设局哭提出了一些规范，这些规范被称为范式

  • 第一范式(IF1): 列不可拆分
  • 第二范式(2NF): 唯一标识
  • 第三范式(3NF): 引用主键
  • 说明：后一个范式，都是在前一个范式的基础上建立的

MySQL 数据类型

MySQL支持多种类型，这里主要列出来常用几种

  • 数字
    • int 整数
    • decimal 小数
  • 字符串
    • char 定长字符串
    • varchar 变长字符串
    • text 长文本数据
  • 日期
    • datetime 混合日期和时间值，时间戳
  • 布尔
    • bit

约束

  • 主键约束 primary key 主键不能重复
  • 非空约束 not null 比如要求姓名不能为空
  • 唯一约束 unique 主键约束只能有一个，唯一约束可以有多个
  • 默认约束 default 比如默认性别为男
  • 外键约束 foreign key

NP-completeness

NP stands for None-deterministic Polynomial. This means that the problem can be solved in Polynomial time using a None-deterministic Turing machine.

Polynomial-Time Solvability

A problem is polynomial-time solvable if there is an algorithm that correctly solves it in \(O(n^k)\) time, for some constant \(k\).

Definition: P = the set of polynomial-time solvable problems.
