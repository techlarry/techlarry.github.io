valgrind

Valgrind(/ˈvælɡrɪnd/) is an instrumentation framework for building dynamic analysis tools. There are Valgrind tools that can automatically detect many memory management and threading bugs, and profile your programs in detail. You can also use Valgrind to build new tools.

The Valgrind distribution currently includes six production-quality tools:

  • a memory error detector
  • two thread error detectors
  • a cache and branch-prediction profiler
  • a call-graph generating cache and branch-prediction profiler
  • a heap profiler.

It also includes three experimental tools:

  • a stack/global array overrun detector,
  • a second heap profiler that examines how heap blocks are used
  • a SimPoint basic block vector generator.

Installing on Mac

The simplest way to install valgrind is using brew:

brew install --HEAD valgrind

Demo

Writing a C program named 'arrays.c' without freeing the memory after allocating it dynamically using malloc, for example,

int* heap_array = (int*) malloc(sizeof(int) * 5);
free(heap_array); ## deleting the statement...

After running by command

valgrind ./arrays

you will see "definitely lost: 20 bytes in 1 blocks!!"

Python汇编代码

学完了汇编语言之后，非常好奇Python程序产生的汇编代码。其实Python作为一种解释语言，其源代码是直接被CPython编译器编译成字节码(btyecode)的；然后在虚拟机中依次解释执行字节码所对应的机器码(machine code)。当需要重用时，字节码以.pyc和.pyo文件存在。当再次运行代码时，虚拟机直接运行.pyc和.pyo文件，省去了重新编译的过程。

Python提供了dis模块，在代码运行时加入-m dis就可以得到源代码对应的汇编代码。其实dis在这里反汇编了字节码。

也可以在源代码中直接调用dis模块：

import dis
def f(x):
    return 1 < x ** 2 < 100

dis.dis(f)

得到的字节码指令(byte code instruction)如下所示：

   4           0 LOAD_CONST               1 (1)
              3 LOAD_FAST                0 (x)
              6 LOAD_CONST               2 (2)
              9 BINARY_POWER
             10 DUP_TOP
             11 ROT_THREE
             12 COMPARE_OP               0 (<)
             15 JUMP_IF_FALSE_OR_POP    25
             18 LOAD_CONST               3 (100)
             21 COMPARE_OP               0 (<)
             24 RETURN_VALUE
        >>   25 ROT_TWO
             26 POP_TOP
             27 RETURN_VALUE

字节码指令的含义可以通过官方文件查找。

OS161

OS161是哈佛大学推出的一款用于教学的操作系统。它运行在一个模拟器上，与BSD类似。目前的版本是2.x版本，支持多处理器和其他现代特性。

OS161 layout

首先运行kernel

sys161 kernel

像正常一样运行gdb

gdb kernel

在完成加载后，在gdb中输入以下内容使其连接到System/161

target remote unix:.sockets/gdb

Character encoding

A character encoding (字符编码) is used to represent a repertoire of characters by some kind of encoding system.

Terminology

  • character: a minimal unit of text that has semantic value.
  • character set: a collection of characters that might be used by multiple languages.
  • coded character set: a character set in which each character corresponds to a unique number.
  • code unit: a bit sequence used to encode each character of a repertoire within a given encoding form.
  • code point(码位): any legal value in the character set.
  • character repertoire(字符集): an abstract list of more than one million characters found in a wide variety of scripts including Latin, Cyrillic, Chinese, Korean, Japanese, Hebrew, and Aramaic.

Unicode

unicode is a computing industry standard for the consistent encoding, representation, and handling of text expressed in most of the world's writing systems. The Unicode standard defines UTF-8, UTF-16, and UTF-32, and several other encodings are in use.

UTF-8

UTF-8, the most widely used by websites. The encoding is variable-length and uses 8-bit code units. UTF-8 encodes each of the 1,112,064 valid code points in Unicode using 1 to 4 8 bit bytes.
